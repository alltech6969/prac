Practical No. 5 
Aim: Installation and Configuration of virtualization using KVM. 
1.	Download Ubuntu 22.04 image from https://www.osboxes.org/ubuntu/#ubuntu-22-04jammy-vbox 
2.	Launch the image in Virtual box. 
3.	Launch Terminal and execute the following code. 
4.	Update and Upgrade Ubuntu 22.04  sudo apt update  sudo apt upgrade  
5.	Check if Virtualization is enabled  egrep -c '(vmx|svm)' /proc/cpuinfo  
6.	Install the cpu-checker package  sudo apt install -y cpu-checker  
7.	verify if KVM virtualization is enabled  kvm-ok  
8.	Install KVM on Ubuntu 22.04  sudo apt install -y qemu-kvm virt-manager libvirt-daemon-system virtinst libvirtclients bridge-utils  
9.	Enable the virtualization daemon  sudo systemctl enable --now libvirtd sudo systemctl start libvirtd  
10.	Check virtualization daemon is running  sudo systemctl status libvirtd  
11.	Add Your User to the KVM and Libvirt Group  sudo usermod -aG kvm $USER  sudo usermod -aG libvirt $USER 
12.	Restart the machine. 
13.	Search for Virtual Machine manager and launch it.

Practical No. 7
Aim: Implement FOSS-Cloud Functionality VSI (Virtual Server Infrastructure) Infrastructure as a Service (IaaS), Storage. 
Steps: 
Install FOSS-Cloud on VM 
1.	Download FOSS ISO file from https://sourceforge.net/projects/fosscloud/ 
2.	Create a virtual machine with 5GB of RAM,150GB Disk space and VT-x enabled. 
3.	Turn on the virtual machine and boot into FOSS installation setup using iso file. 
4.	Select the first option that is “default boot option”. 
5.	Type yes to start the installation. 
6.	Select Node type as 1. Demo-System. 
7.	Select the disk usually its “sda”. 
8.	Type yes and enter(this will wipe the data on the disk) 
9.	Enter the available ethernet device name. 
10.	Type yes to enable DHCP. 
11.	Type yes to allow the system to reboot. 
12.	Login using the credentials provided. 
13.	Enter the following command to configure the server “fc-nodeconfiguration -n demo-system –password admin”. 
14.	Type the ifconfig command to get the ip address of the FOSS Server. 
15.	Launch any browser on any other machine and enter the ip address. 
16.	Login using the provided credentials. 
17.	Install virt-viewer and spice client on your machine 
18.	Press windows + R and type regedit. 
19.	Goto HKEY_CLASSES_ROOT -> spice -> shell -> open -> command 
20.	Goto this path C:/Users/Admin/AppData/Local/virt-viewer/bin/remoteviewer 
21.	Come back to command right click in Default > edit and paste the path 
“C:/Users/Admin/AppData/Local/virt-viewer/bin/remote-viewer %1” 
Creating a Virtual Machine 
1.	Virtual Machine > Profile > Upload ISO File 
2.	Choose File > Give it a name > click on Upload 
3.	Virtual Machine > Profile > Create 
4.	BaseProfile > windows > x86 > en-US > select the ISO file > give VM the name > 2GB ram > 10GB storage > 1CPU > localtime > Create button. 
Creating VM template 
1.	Virtual Machine > VM templates > create  
2.	Keeps things default choose vmpool and no. of displays as 1. 
3.	Click create 
4.	Click Run button in Run Action button(VM status will change to running) 
5.	Click on display(4th from left) under action section 
Using VM 
1.	Click > Try Ubuntu. 
2.	Run Ubuntu on Cloud.

Practical No. 10 
Aim: Implementation of Openstack with user and private network creation. 
Steps: 
Note: The commands are underlined. 
1.	Download Ubuntu 22.04 image from https://www.osboxes.org/ubuntu/#ubuntu-22-04jammy-vbox 
2.	Launch the image in Virtual box. 
3.	Launch Terminal and execute the following code. 
4.	sudo useradd -s /bin/bash -d /opt/stack -m stack 
This code will create a new user stack with its home directory being /opt/stack 
5.	sudo chmod +x /opt/stack 
Adding execute permission to /opt/stack 
6.	echo "stack ALL=(ALL) NOPASSWD: ALL" | sudo tee /etc/sudoers.d/stack 
7.	sudo -u stack -i the commands 6 & 7 makes sure that stack user has sudo previlages and doesnot have to enter password again and again. 
8.	git clone https://opendev.org/openstack/devstack 
9.	cd devstack 
Downloading devstack and changing directory to devstack 10.  Now we have create a file and add the following lines to it. 
11.	nano local.conf  
(this will create and open the local.conf file) type the whole text below in-it. 
 [[local|localrc]] 
ADMIN_PASSWORD=secret 
DATABASE_PASSWORD=$ADMIN_PASSWORD 
RABBIT_PASSWORD=$ADMIN_PASSWORD 
SERVICE_PASSWORD=$ADMIN_PASSWORD 
12.	After typing the whole test. Press “Ctrl + O” to save the file then click Enter to confirm the file name. 
13.	Now press “Ctrl + X” to exit. 
14.	./stack.sh 
This command will start the install. 
OR 
Execute the below command in latest Ubuntu LTS to install openstack. 1. sudo snap install openstack  
2.	sunbeam prepare-node-script | bash -x && newgrp snap_daemon  
3.	sunbeam cluster bootstrap --accept-defaults  
4.	sunbeam configure --accept-defaults --openrc demo-openrc  
5.	sunbeam launch ubuntu -n test
